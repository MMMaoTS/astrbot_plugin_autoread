# BACKUP

AutoRead 备份恢复功能，用于保护书籍和笔记数据。

## 功能概述

- 书籍和笔记分开备份
- 备份格式为 zip 包
- 导入前预览
- 合并导入（新 ID 导入，已有 ID 跳过）
- 导入历史记录

## 导出

进入「备份恢复」页面，在「数据导出」区域：

- **导出书籍**：打包 books/、chunks/ 和 book_index.json
- **导出笔记**：打包 notes/ 和 records_index.json
- **导出完整备份**：打包所有数据（books、chunks、notes、state.json）

下载格式为 zip 文件。

## 导入

导入分两步：

### 1. 解析备份

选择一个 .zip 备份文件，点击「解析备份」。系统会预览：

- 备份 ID
- 备份类型
- 总记录数
- 可导入数
- 已有 ID 数量（会自动跳过）

### 2. 合并导入

预览确认后，点击「合并导入」。系统会：

- 导入新书籍（book_id 已存在则跳过）
- 导入新笔记（record_id 已存在则跳过）
- 记录导入历史

## 导入规则

- 只按 ID 去重，不检查内容是否相同
- 已导入过的 backup_id 默认跳过
- 不支持覆盖恢复、强制覆盖或清空恢复
- 导入模式固定为合并（merge）
- 以上规则仅适用于备份恢复，不影响其他模块

## 导入历史

每次成功导入都会记录在「导入记录」表格中，包含备份 ID、类型、导入数量、跳过数量和状态。

## 注意事项

- 备份恢复不会覆盖当前插件配置
- 书籍管理页和普通上传不受备份规则影响
- 笔记阅览页仍为只读
- 备份文件保存在 plugin_data 目录下
