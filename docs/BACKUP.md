# BACKUP

AutoRead 备份恢复功能，用于保护书籍和笔记数据。

## 功能概述

- 书籍和笔记分开备份
- 备份格式为 zip 包
- 导入前预览
- 合并导入（新 ID 导入，已有 ID 跳过）
- 导入历史记录
- 服务器备份文件管理（查看、下载、恢复、删除）
- 上传备份文件到服务器

## 导出

进入「备份恢复」页面，在「数据导出」区域：

- **导出书籍**：打包 books/、chunks/ 和 book_index.json
- **导出笔记**：打包 notes/ 和 records_index.json
- **导出完整备份**：打包所有数据（books、chunks、notes、state.json）

导出的备份同时保存到服务器 `backups/` 目录，文件名格式：

```
autoread_backup_{type}_{YYYYMMDD_HHMMSS}_{random}.zip
```

例如：`autoread_backup_full_20260624_142530_a1b2c3.zip`

下载格式为 zip 文件，可通过浏览器下载按钮保存到本地。

## 服务器备份管理

WebUI「备份恢复」页面底部有「服务器备份管理」区域。

### 查看备份文件

列出服务器 `backups/` 目录下所有 .zip 备份文件，显示文件名、大小、修改时间、来源（导出/上传）。

### 下载备份文件

点击列表中的「下载」按钮即可下载对应备份文件到本地。下载不修改服务器数据，无需确认。

### 上传备份到服务器

1. 选择本地 .zip 备份文件。
2. 点击「上传到服务器」。
3. 文件保存到 `backups/` 目录，文件名格式：`autoread_uploaded_{YYYYMMDD_HHMMSS}_{original_name}.zip`
4. 上传后不自动恢复，需手动点击「恢复」。

### 从服务器备份恢复

1. 在备份列表中点击某个文件的「恢复」。
2. 系统**自动解析**备份文件（调用 inspect API），展示备份类型、书籍/笔记数量、恢复模式。
3. 确认弹窗 → 确认后执行合并导入恢复。
4. 恢复后刷新书籍、笔记、任务列表。
5. 不需要用户手动点击"解析"按钮。

### 删除服务器备份

1. 在备份列表中点击「删除」。
2. 确认弹窗 → 确认后删除服务器上的备份文件。
3. 删除不可逆。

## 导入

导入分两步：

### 1. 解析备份

选择一个 .zip 备份文件，点击「解析备份」。系统会预览：

- 备份 ID
- 备份类型
- 总记录数
- 可导入数
- 已有 ID 数量（会自动跳过）

### 2. 合并导入

预览确认后，点击「合并导入」。系统会：

- 导入新书籍（book_id 已存在则跳过）
- 导入新笔记（record_id 已存在则跳过）
- 记录导入历史

## 导入规则

- 只按 ID 去重，不检查内容是否相同
- 已导入过的 backup_id 默认跳过
- 不支持覆盖恢复、强制覆盖或清空恢复
- 导入模式固定为合并（merge）
- 以上规则仅适用于备份恢复，不影响其他模块

## 导入历史

每次成功导入都会记录在「导入记录」表格中，包含备份 ID、类型、导入数量、跳过数量和状态。

## 笔记恢复说明

- 删除笔记是永久删除当前数据中的记录，不进入回收站。
- 如果误删笔记，可通过备份恢复找回（前提是之前导出过包含该笔记的备份）。
- 备份恢复是整包合并导入，按 record_id 去重。
- 插件不提供手工新增/编辑笔记的 WebUI 入口。
- 重新阅读是重新生成内容的方式，不等同于恢复旧笔记。

## 注意事项

- 备份恢复不会覆盖当前插件配置
- 书籍管理页和普通上传不受备份规则影响
- 备份文件保存在 `plugin_data/astrbot_plugin_autoread/backups/` 目录下
- 旧格式备份文件仍兼容，可正常查看、下载、恢复、删除
- 建议定期清理旧备份文件，避免磁盘堆积
